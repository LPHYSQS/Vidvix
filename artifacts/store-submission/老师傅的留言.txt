# 一、你这个工作流的问题本质（重点）

👉 你现在的流程是：

```
dotnet publish → 手写 AppxManifest → makeappx 封 MSIX
```

这个做法的问题是：

> ❌ **绕过了 WinUI 3 官方打包系统（Windows App SDK）**

---

### 会导致什么？

你现在遇到的两个现象刚好是典型症状：

---

## ❗问题1：启动闪退

原因通常是：

* WinUI 3 运行依赖没声明（Windows App SDK framework）
* 启动模型错误（Win32 → UWP 包化不完整）
* DeploymentManager 初始化冲突（你已经抓到了）

👉 简单说：

> **包“长得像 MSIX”，但内部不是一个“合法的 WinUI 3 包应用”**

---

## ❗问题2：多出来一个“无图标副进程”

这个更关键 👇

👉 说明：

* 你真正运行的是一个“桥接进程 / stub”
* 主程序没有被正确注册为 App Entry

通常是：

```xml
<Application Executable="xxx.exe" EntryPoint="Windows.FullTrustApplication">
```

但：

* 路径错 ❌
* 或 WinUI 3 没走正确启动链 ❌

---

👉 在 MSIX 里：

# ✅ 方案：改回官方 Single-project MSIX（强烈建议）

---

## 第一步：补 Package.appxmanifest（关键）

在项目根目录创建：

👉 `Package.appxmanifest`

最小可用结构（我给你精简版）：

```xml
<?xml version="1.0" encoding="utf-8"?>
<Package
  xmlns="http://schemas.microsoft.com/appx/manifest/foundation/windows10"
  xmlns:uap="http://schemas.microsoft.com/appx/manifest/uap/windows10"
  xmlns:rescap="http://schemas.microsoft.com/appx/manifest/foundation/windows10/restrictedcapabilities">

  <Identity
    Name="你的应用ID"
    Publisher="CN=你的证书"
    Version="1.0.0.0" />

  <Properties>
    <DisplayName>你的应用名</DisplayName>
    <PublisherDisplayName>你的名字</PublisherDisplayName>
    <Logo>Assets\StoreLogo.png</Logo>
  </Properties>

  <Dependencies>
    <TargetDeviceFamily Name="Windows.Desktop" MinVersion="10.0.19041.0" MaxVersionTested="10.0.22621.0" />
  </Dependencies>

  <Applications>
    <Application Id="App"
      Executable="$targetnametoken$.exe"
      EntryPoint="Windows.FullTrustApplication">
      <uap:VisualElements
        DisplayName="你的应用名"
        Square150x150Logo="Assets\Square150x150Logo.png"
        Square44x44Logo="Assets\Square44x44Logo.png"
        Description="描述"
        BackgroundColor="transparent"/>
    </Application>
  </Applications>

  <Capabilities>
    <rescap:Capability Name="runFullTrust" />
  </Capabilities>

</Package>
```

---

## 第二步：修改 `.csproj`

加上：

```xml
<PropertyGroup>
  <WindowsPackageType>MSIX</WindowsPackageType>
  <GenerateAppxPackageOnBuild>true</GenerateAppxPackageOnBuild>
</PropertyGroup>
```

---

## 第三步：禁用 DeploymentManager（你这个很关键）

找到：

```xml
WindowsAppSdkDeploymentManagerInitialize
```

👉 改成：

```xml
false
```

或者直接删掉

---

## 第四步：重新打包（官方方式）

```bash
dotnet publish -c Release -r win-x64 ^
  -p:UapAppxPackageBuildMode=StoreUpload
```

---

👉 你会得到：

* `.msix`
* `.msixupload`（用于上传 Microsoft Partner Center）
